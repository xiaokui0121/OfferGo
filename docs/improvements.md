# OfferGo 改进待办

## 广告/引流帖辨别（待解决）

### 问题
小红书上有一类「伪装成招聘帖」的广告/引流内容，特征：
- 同一博主短时间内连续发多条「招聘帖」，话术高度雷同
- 不写具体公司名（用「某公司」「某 AI 初创」「一家大厂」代替）
- 投递方式只留私信/V 信而不是邮箱
- 实际目的：导流到付费简历优化、面试辅导、内推服务、社群

### 用户辨别方式
点进博主主页 → 看到 ta 发了一堆话术雷同的「招聘帖」→ 判定是广告。

### 已实现的初步过滤（v1）
- prompt 强制要求公司名为实体名称（字节、Moonshot、Apexmind 这种）
- 后端入库前一票否决：company 含「某」、纯泛指（「AI 初创公司」「上海某 AI」「本团队」）、N/A、null 一律丢
- 提取出公司名后用正则黑名单二次验证

### 未来可做的（按成本 / 收益排序）

1. **博主维度去重**（最容易、最有效）
   - 入库时记录 note_author，已有字段
   - 同一 author 短时间内（如 7 天）入库 N 条以上时，整段 author 拉黑或在卡片打红色「⚠ 该博主疑似引流」标记
   - UI 加「不再采集该博主」按钮，写入黑名单表

2. **话术雷同检测**
   - 入库前对 jd_raw 做 simhash / minhash，与最近 50 条比对
   - 相似度 > 0.85 直接拒收（基本是同一篇换皮）

3. **投递方式启发式**
   - 只留私信/V 信（无邮箱、无公司链接、无正式招聘平台）+ 公司名模糊 → 拒收
   - 留邮箱（@公司域名）→ 强信号通过

4. **博主主页签名抓取**
   - 进博主主页拉简介，简介里含「简历优化」「面试辅导」「内推 1v1」「付费咨询」「VIP」直接拉黑

5. **用户反馈循环**
   - 详情页加「这是广告」按钮，点了之后：
     - 该 author 自动加入黑名单
     - 删除该岗位
     - 后续过滤模型可用这些样本微调（暂不做）

### 优先级建议
建议先做 **1（博主维度去重 / 拉黑）**——成本最低，覆盖了用户描述的核心识别路径。
